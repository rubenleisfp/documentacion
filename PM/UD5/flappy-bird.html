Flappy Bird en Godot 4

En esta actividad vas a montar paso a paso un clon de Flappy Bird desde cero, entendiendo cada pieza del proyecto: escenas, scripts, físicas, señales y GUI.

Motor: Godot 4.x Nivel: Principiante Duración aprox.: 3–4 horas Lenguaje: GDScript

¿Qué vas a aprender?

Al terminar esta actividad habrás trabajado los conceptos fundamentales de Godot 4: árbol de nodos, escenas reutilizables, CharacterBody2D, físicas 2D, señales, Autoload (Global), Timer, Area2D, CanvasLayer para la interfaz y cómo reiniciar una partida.

💡 Concepto clave — Escena vs Script: Una escena (.tscn) define la estructura visual y los nodos (como un "molde"). Un script (.gd) añade la lógica (el comportamiento) a esa estructura. Casi siempre van de la mano.
Pasos de la Actividad
01

Crea y configura el proyecto

Crear el proyecto nuevo en Godot 4 con las dimensiones correctas

1.1
Abre Godot 4 y crea un nuevo proyecto.

Dale el nombre flappy-bird. Elige una carpeta vacía. En "Renderer" selecciona Compatibility (más ligero, ideal para 2D).

1.2
Ajusta el tamaño de ventana.

Ve a Proyecto → Configuración del Proyecto → Display → Window. Pon el ancho en 288 y el alto en 512. En Stretch → Mode selecciona canvas_items para que escale bien.

1.3
Limita los FPS a 60.

En la misma ventana, busca Application → Run y pon max_fps = 60. Esto evita que el juego vaya más rápido de lo esperado.

1.4
Importa los assets (arte y sonidos).

Copia la carpeta art/ del proyecto original dentro de tu carpeta de proyecto. Godot la detectará automáticamente en el FileSystem.

💡 ¿Por qué 288×512? Son las dimensiones originales del Flappy Bird para móvil. Con el modo canvas_items, Godot escala todo automáticamente sin que tengas que preocuparte por distintos tamaños de pantalla.
02

Crea el script Global (Autoload)

El cerebro central del juego: puntuación, estado y señales

Un Autoload en Godot es un nodo/script que siempre está activo y accesible desde cualquier parte del juego, sin tener que buscarlo en el árbol de escenas. Lo usamos para compartir el estado del juego (puntuación, ¿ha empezado?, etc.).

2.1
Crea la carpeta scripts/ en el FileSystem.

Clic derecho en res:// → Nueva carpeta → scripts.

2.2
Crea el archivo global.gd dentro de scripts/.

Clic derecho en la carpeta → Nuevo Script → nómbralo global.gd. Borra el contenido por defecto y escribe el siguiente código:

GDScript# global.gd — Autoload: compartido en todo el juego
extends Node

# Señales: eventos que otros nodos pueden escuchar
signal on_increment_score
signal on_game_start
signal on_game_over

# Variables de estado
var score := 0
var is_start := false

func game_start():
    is_start = true
    score = 0
    emit_signal("on_increment_score")
    emit_signal("on_game_start")

func increment_score():
    score += 1
    emit_signal("on_increment_score")

func game_over():
    is_start = false
    get_tree().paused = true
    emit_signal("on_game_over")
2.3
Registra el script como Autoload.

Ve a Proyecto → Configuración del Proyecto → Autoload. Pulsa el icono de carpeta, selecciona scripts/global.gd y asegúrate de que el nombre es Global. Pulsa "Añadir".

💡 ¿Qué es una señal (signal)? Una señal es como un botón de aviso. Cuando algo pasa (por ejemplo, el jugador puntúa), Global emite la señal y cualquier nodo que esté "escuchando" reacciona. Así los nodos no se llaman directamente entre sí y el código queda más limpio.
03

Crea la escena del Pájaro

El protagonista: físicas, salto, rotación y animación

3.1
Crea una nueva escena y elige CharacterBody2D como nodo raíz.

Escena → Nueva Escena → Selecciona "Otros Nodos" → busca CharacterBody2D. Este tipo de nodo tiene soporte nativo para físicas, colisiones y el método move_and_slide().

3.2
Renombra el nodo raíz a Bird y añade los nodos hijos:

Selecciona el nodo raíz y añade (botón "+") estos hijos:

  • AnimatedSprite2D → para la animación de aleteo
  • CollisionShape2D → para detectar colisiones (forma circular)
  • AudioStreamPlayer → nombrar AudioFly, asignar art/audio/wing.wav
📦 Bird [CharacterBody2D] ← bird.gd ├── AnimatedSprite2D [AnimatedSprite2D] ├── CollisionShape2D [CollisionShape2D] └── AudioFly [AudioStreamPlayer]
3.3
Configura la animación en AnimatedSprite2D.

Selecciona el nodo → en el Inspector, en Sprite Frames, crea un nuevo recurso SpriteFrames. Añade una animación llamada fly con los 3 fotogramas del pájaro (redbird-downflap, midflap, upflap). Pon la velocidad a 10 fps y activa "Loop". Asigna la animación activa a fly.

3.4
Configura la colisión.

Selecciona CollisionShape2D → en el Inspector, crea una nueva CircleShape2D. Ajusta el radio hasta que rodee al sprite del pájaro.

3.5
Crea el script bird.gd y asígnalo al nodo raíz Bird. 
GDScriptextends CharacterBody2D
class_name Bird

const JUMP_VELOCITY = -300.0

func _physics_process(delta: float) -> void:
    if Global.is_start:
        if not is_on_floor():
            # Gravedad: cuanto más tiempo cae, más rápido va
            velocity += get_gravity() * delta
        
        if Input.is_action_just_pressed("ui_accept"):
            velocity.y = JUMP_VELOCITY
            rotation = deg_to_rad(-30)
            $AudioFly.play()
    
    move_and_slide()
    rotation_bird()

func rotation_bird():
    # Inclina el pájaro hacia abajo mientras cae
    if velocity.y > 0 and rad_to_deg(rotation) < 90:
        rotation += 2 * deg_to_rad(1)
    elif velocity.y > 0 and rad_to_deg(rotation) > -30:
        rotation -= 2 * deg_to_rad(1)
3.6
Guarda la escena como scenes/bird.tscn.
💡 ¿Qué hace _physics_process? Es una función especial que Godot llama automáticamente 60 veces por segundo, sincronizada con el motor de físicas. Siempre usa el parámetro delta para que el movimiento sea igual independientemente de la velocidad del ordenador.
⚠️ class_name Bird es importante. Al escribir class_name Bird, Godot registra este script con ese nombre en todo el proyecto. Así otros scripts pueden hacer if body is Bird para comprobar si el objeto que colisiona es el pájaro.

Un obstáculo simple reutilizable como bloque de construcción

Renómbralo Pipe. Area2D detecta solapamientos pero no tiene físicas de cuerpo sólido; perfecto para detectar si el pájaro toca la tubería.

4.2

05

Crea la escena del Par de Tuberías (Pipe_game)

Dos tuberías + zona de puntos + lógica de movimiento y colisión

5.1
Nueva escena con nodo raíz Node2D.

Renómbralo Pipe_game. Node2D es el nodo base para cualquier objeto 2D sin físicas propias.

5.2
Instancia la escena pipe.tscn dos veces como hijos.

Clic derecho en Pipe_game → "Instanciar escena hijo" → selecciona pipe.tscn. Repite. Renómbralos PipeDown (la de arriba, girada 180°) y PipeTop (la de abajo).

Para PipeDown: Rotation = 180° (3.1416 rad), posición Y = -224.
Para PipeTop: posición Y = 224.

5.3
Añade un VisibleOnScreenNotifier2D en posición X=120.

Este nodo emite una señal cuando sale de la pantalla. Lo usaremos para borrar la tubería y ahorrar memoria.

5.4
Añade un Area2D hijo llamado AreaAddPoint.

Dentro, añade un CollisionShape2D con un rectángulo fino (20×144 px) en posición X=64, entre las dos tuberías. Esta área detectará cuando el pájaro pase por el hueco.

5.5
Añade dos AudioStreamPlayer2D:
  • AudioGameOver → asignar hit.wav
  • AudioPoint → asignar point.wav
📦 Pipe_game [Node2D] ← pipe_game.gd ├── AudioGameOver [AudioStreamPlayer2D] ├── AudioPoint [AudioStreamPlayer2D] ├── PipeDown [instancia de pipe.tscn] rot: 180° ├── PipeTop [instancia de pipe.tscn] ├── VisibleOnScreenNotifier2D └── AreaAddPoint [Area2D] └── CollisionShape2D
5.6
Crea el script pipe_game.gd y asígnalo al nodo raíz. 
GDScriptextends Node2D

var velocity = 200  # velocidad de desplazamiento

func _ready() -> void:
    position.x = 330          # empieza fuera de la pantalla (derecha)
    position.y = randi_range(133, 300)  # altura aleatoria

func _process(delta: float) -> void:
    position.x -= delta * velocity  # mueve a la izquierda

# Cuando la tubería sale de la pantalla
func _on_visible_on_screen_exit() -> void:
    queue_free()  # destruye el nodo para liberar memoria

# Si el pájaro toca la tubería
func _on_pipe_bird(body: Node2D) -> void:
    if body is Bird:
        Global.game_over()
        $AudioGameOver.play()

# Si el pájaro pasa por el hueco
func _on_area_add_point_body_entered(body: Node2D) -> void:
    if body is Bird:
        Global.increment_score()
        $AudioPoint.play()
5.7
Conecta las señales.

Selecciona PipeDown → pestaña "Nodo" (señales) → body_entered → conecta a _on_pipe_bird en Pipe_game.
Repite para PipeTop.
Selecciona VisibleOnScreenNotifier2D → señal screen_exited → conecta a _on_visible_on_screen_exit.
Selecciona AreaAddPoint → señal body_entered → conecta a _on_area_add_point_body_entered.

5.8
Guarda como scenes/pipe_game.tscn.
💡 queue_free() vs free() queue_free() le pide a Godot que elimine el nodo de forma segura al final del frame actual. Usar free() directamente puede causar errores si el nodo todavía está procesando algo. Usa siempre queue_free().
06

Crea la interfaz gráfica (GUI)

Puntuación, pantalla de inicio y Game Over

6.1
Nueva escena con nodo raíz CanvasLayer.

Renómbralo GUI. CanvasLayer dibuja siempre por encima del juego y no se ve afectado por la cámara ni por el scroll del fondo.

6.2
Añade estos nodos hijos:
  • Label → renómbralo LabelScore. Texto inicial: 0. Fuente: art/flappy-font.ttf, tamaño 50. Posición centrada en la parte superior.
  • Sprite2D → renómbralo Message. Textura: art/sprites/message.png. Posición central.
  • Sprite2D → renómbralo Gameover. Textura: art/sprites/gameover.png. Posición central.
📦 GUI [CanvasLayer] ← gui.gd ├── LabelScore [Label] ├── Message [Sprite2D] └── Gameover [Sprite2D]
6.3
Crea el script gui.gd y asígnalo al nodo GUI. 
GDScriptextends CanvasLayer

func _ready() -> void:
    Global.connect("on_increment_score", change_score)
    Global.connect("on_game_over", game_over)
    $Gameover.hide()
    $Message.show()
    get_tree().paused = false

# Detecta pulsaciones de teclado/ratón
func _input(event: InputEvent) -> void:
    if Input.is_action_just_pressed("ui_accept"):
        if Global.is_start == false:
            Global.game_start()
            game_start()

func game_start():
    $Message.hide()

func game_over():
    $Gameover.show()
    # Espera 2 segundos antes de reiniciar
    await get_tree().create_timer(2).timeout
    get_tree().reload_current_scene()

func change_score():
    $LabelScore.text = str(Global.score)
6.4
Guarda como scenes/gui.tscn.
💡 ¿Qué hace await? await pausa la ejecución de la función hasta que ocurra el evento indicado. En este caso espera 2 segundos (create_timer(2).timeout) sin bloquear el resto del juego. Es la forma moderna de hacer un "espera X segundos y luego..." en Godot 4.
07

Monta la escena principal (Level)

El punto de entrada del juego: fondo, pájaro, suelo y generador de tuberías

7.1
Nueva escena con nodo raíz Node.

Renómbralo Level. Un Node básico es suficiente para el nodo raíz ya que no necesita propiedades 2D directas.

7.2
Añade el fondo con scroll automático.

Añade un nodo Parallax2D y renómbralo ParallaxBackground. En el Inspector: repeat_size = Vector2(288, 0) y autoscroll = Vector2(-30, 0). Dentro, añade un Sprite2D con la textura background-day.png en posición (144, 256).

7.3
Instancia la GUI dentro de ParallaxBackground.

Clic derecho en ParallaxBackground → "Instanciar escena hijo" → selecciona gui.tscn.

7.4
Añade el suelo animado.

Añade otro Parallax2D hijo del nodo Level (no de ParallaxBackground), llámalo ParallaxFloor. repeat_size = Vector2(336, 0), autoscroll = Vector2(-60, 0). Dentro, un Sprite2D con base.png en posición (288, 464), escala X = 1.71.

7.5
Instancia el pájaro.

Clic derecho en Level → "Instanciar escena hijo" → selecciona bird.tscn. Ponlo en posición (120, 112) y z_index = 1 (para que se dibuje por encima del fondo).

7.6
Añade el Timer para generar tuberías.

Añade un nodo Timer hijo de Level, llámalo TimerPipe. En el Inspector, wait_time = 1.5 segundos.

7.7
Añade la zona de detección del suelo.

Añade un nodo Area2D → AreaFloor. Dentro, un CollisionShape2D con un rectángulo (572×64) centrado en posición (290, 440), justo sobre el suelo visible.

7.8
Añade el sonido de Game Over.

Añade un AudioStreamPlayer2D → AudioGameOver → asigna hit.wav.

📦 Level [Node] ← level.gd ├── AudioGameOver [AudioStreamPlayer2D] ├── ParallaxBackground [Parallax2D] │ ├── BackgroundDay [Sprite2D] │ └── GUI [instancia gui.tscn] ├── ParallaxFloor [Parallax2D] │ └── Base [Sprite2D] ├── Bird [instancia bird.tscn] ├── TimerPipe [Timer] └── AreaFloor [Area2D] └── CollisionShape2D
7.9
Crea el script level.gd y asígnalo al nodo Level. 
GDScriptextends Node

# Variable exportada: asignable desde el editor
@export var pipe_game_scene: PackedScene

func _ready() -> void:
    Global.connect("on_game_start", game_start)

func game_start():
    $TimerPipe.start()

func create_pipe():
    var pipe_game = pipe_game_scene.instantiate()
    add_child(pipe_game)

# El Timer genera una tubería cada 1.5 segundos
func _on_timer_pipe_timeout() -> void:
    create_pipe()

# Si el pájaro toca el suelo
func _on_area_floor_body_entered(body: Node2D) -> void:
    if body is Bird:
        Global.game_over()
        $TimerPipe.stop()
        $AudioGameOver.play()
7.10
Asigna la escena de la tubería a la variable exportada.

Selecciona el nodo Level en el árbol → en el Inspector verás Pipe Game Scene → arrastra pipe_game.tscn desde el FileSystem hasta esa ranura.

7.11
Conecta las señales del Timer y del AreaFloor.

Selecciona TimerPipe → señal timeout → conecta a _on_timer_pipe_timeout en Level.
Selecciona AreaFloor → señal body_entered → conecta a _on_area_floor_body_entered.

7.12
Guarda como scenes/level.tscn y márcala como escena principal.

Ve a Proyecto → Configuración del Proyecto → Application → Run → Main Scene y selecciona scenes/level.tscn.

💡 ¿Qué es @export? El decorador @export hace que una variable aparezca en el Inspector del editor. Así puedes cambiar valores o asignar recursos sin tocar el código. Es muy útil para hacer el juego más configurable.
08

¡Prueba y ajusta el juego!

Corre el proyecto y comprueba que todo funciona correctamente

8.1
Pulsa F5 (o el botón ▶) para ejecutar el proyecto.

Deberías ver el pájaro quieto con el mensaje de inicio. Al pulsar Espacio, el juego comienza.

8.2
Comprueba la lista de verificación:
  • ✅ El fondo y el suelo se mueven solos
  • ✅ Al pulsar Espacio el pájaro salta
  • ✅ El pájaro cae por gravedad
  • ✅ Salen tuberías cada 1.5 segundos a distintas alturas
  • ✅ Al pasar por el hueco sube la puntuación
  • ✅ Al tocar una tubería o el suelo → Game Over
  • ✅ Tras 2 segundos el juego se reinicia
⚠️ Problema común: el pájaro cae antes de pulsar Espacio. Revisa que en bird.gd la gravedad esté dentro del bloque if Global.is_start:. Si no, el pájaro caerá incluso en la pantalla de inicio.
⚠️ Las tuberías no aparecen. Comprueba que en el Inspector del nodo Level la variable Pipe Game Scene tiene asignada la escena pipe_game.tscn. Si está vacía, arrástrala desde el FileSystem.
