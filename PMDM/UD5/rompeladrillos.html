Rompe Ladrillos en Godot 4

Recrea paso a paso el clásico Breakout: pala, bola con rebotes físicos, ladrillos con animación de destrucción, puntuación y gestión de partida.

Motor: Godot 4.x Nivel: Principiante Duración aprox.: 3–4 horas Resolución: 1280×720

¿Qué vas a aprender?

Trabajarás con CharacterBody2D y StaticBody2D, rebotes físicos con move_and_collide(), grupos de nodos, animaciones con Tween, dibujo 2D personalizado con _draw(), Marker2D para posicionado inicial, Autoload para la puntuación y gestión del ciclo de partida.

02

Crea el script Global (Autoload)

Gestión centralizada de la puntuación

2.1
Crea la carpeta scripts/ y dentro el archivo global.gd. 
GDScript# global.gd — Autoload: puntuación accesible desde todo el juego
extends Node

var score := 0

func increment_score():
    score += 1

func reset_score():
    score = 0
2.2
Registra el script como Autoload.

Proyecto → Configuración → Autoload. Selecciona scripts/global.gd, nombre: Global. Pulsa Añadir.

💡 Diferencia con el proyecto Flappy Bird: En este juego el Global es más sencillo: solo lleva la puntuación. No necesitamos señales porque la actualización del marcador se hace directamente en el _process de main, leyendo Global.score en cada frame.
03

Crea la escena del círculo decorativo (Circule)

Dibujo 2D personalizado con código — la bola visible

En lugar de usar un sprite o una imagen, vamos a dibujar la bola directamente con código usando el método _draw() de Godot. Esta escena se instanciará dentro de la bola.

3.1
Nueva escena con nodo raíz Node2D.

Renómbralo Circule. Guarda como scenes/circule.tscn.

3.2
Crea el script circule.gd y asígnalo. 
GDScript@tool          # permite ver el resultado en el editor sin ejecutar el juego
extends Node2D

var circle_position = Vector2.ZERO
var circle_radius = 8
var circle_color := Color("#fff")

func _draw() -> void:
    # draw_circle(centro, radio, color)
    draw_circle(circle_position, circle_radius, circle_color)
💡 ¿Qué es @tool? El decorador @tool hace que el script se ejecute también dentro del editor de Godot (no solo al jugar). Así puedes ver el círculo dibujado en la escena mientras diseñas, sin tener que pulsar Play. Muy útil para herramientas visuales y depuración.
💡 ¿Qué es _draw()? Es una función especial que Godot llama cuando necesita redibujar el nodo. Dentro puedes usar métodos como draw_circle(), draw_rect(), draw_line(), etc. para crear gráficos completamente por código, sin imágenes externas.
04

Crea la escena de la Bola (Ball)

Movimiento, rebote físico y detección de ladrillos

4.1
Nueva escena con nodo raíz CharacterBody2D.

Renómbralo Ball. En el Inspector, pon Motion Mode en Floating (la bola se mueve en todas las direcciones, no solo vertical como un personaje). Guarda como scenes/ball.tscn.

4.2
Añade los nodos hijos:
  • CollisionShape2D → forma CircleShape2D, radio 8, escala 1.2
  • Instancia de circule.tscn → la representación visual de la bola
📦 Ball [CharacterBody2D · Motion: Floating] ← ball.gd ├── CollisionShape2D [CircleShape2D r=8] └── Circule [instancia circule.tscn]
4.3
Crea el script ball.gd y asígnalo. 
GDScriptextends CharacterBody2D

const SPEED = 300.0

# Función pública: la llama main.gd al iniciar partida
func start() -> void:
    # La bola sale hacia abajo (Vector2(0,1) = dirección abajo)
    velocity = Vector2(0, 1) * SPEED

func _physics_process(delta: float) -> void:
    # move_and_collide mueve la bola y devuelve info de la colisión
    var collision = move_and_collide(velocity * delta)
    
    if collision:
        # ¿Con qué ha colisionado?
        var ref = collision.get_collider() as StaticBody2D
        
        if ref and ref.is_in_group("brick_group"):
            # Es un ladrillo: lo destruimos y sumamos punto
            ref.destroy_brick()
            Global.increment_score()
        
        # Rebote: invierte la dirección usando la normal de la superficie
        velocity = velocity.bounce(collision.get_normal())
💡 move_and_collide() vs move_and_slide()
move_and_slide() (usado en Flappy Bird y en la pala) desliza automáticamente el objeto por las superficies — ideal para personajes que caminan.
move_and_collide() devuelve información detallada de la colisión (normal, punto de contacto) y no desliza. Perfecto para una bola que necesita calcular el ángulo de rebote.
💡 ¿Qué es velocity.bounce(normal)? bounce() calcula el vector reflejado usando la normal de la superficie de impacto. Es el equivalente matemático de un espejo: la bola rebota con el ángulo de incidencia igual al de reflexión. No tienes que calcular ángulos manualmente.

Crea la escena del Ladrillo (Brick)

Obstáculo estático con animación de destrucción mediante Tween

5.1
Nueva escena con nodo raíz StaticBody2D.

Renómbralo Brick. StaticBody2D es un cuerpo físico que no se mueve — perfecto para paredes, suelos y obstáculos.

5.2
Añade los nodos hijos:
  • ColorRect → tamaño 80×16, posición (-40, 8), color amarillo: #ffff30
  • CollisionShape2D → forma RectangleShape2D, tamaño 80×16, posición (0, 16)
5.3
⚠️ Asigna el grupo brick_group al nodo Brick.

Con el nodo Brick seleccionado, ve a la pestaña Nodo → Grupos y añade el grupo brick_group. Esto es fundamental: la bola detecta ladrillos usando is_in_group("brick_group").

📦 Brick [StaticBody2D · grupo: brick_group] ← brick.gd ├── ColorRect [80×16 · color amarillo] └── CollisionShape2D [RectangleShape2D 80×16]
5.4
Crea el script brick.gd y asígnalo. 
GDScriptextends StaticBody2D

func destroy_brick() -> void:
    # 1. Desactiva la colisión inmediatamente (con set_deferred para evitar errores)
    $CollisionShape2D.set_deferred("disabled", true)
    
    # 2. Crea un Tween para animar el desvanecimiento
    var tween: Tween = get_tree().create_tween()
    
    # 3. Anima la propiedad "modulate:a" (transparencia alfa) de 1 a 0 en 0.5 segundos
    tween.tween_property(self, "modulate:a", 0.0, 0.5)
    
    # 4. Cuando termina la animación, ejecuta finish_animation()
    tween.tween_callback(finish_animation)

func finish_animation():
    queue_free()   # elimina el nodo de la escena
💡 ¿Qué es un Tween? Un Tween es un animador de propiedades. Le dices "lleva esta propiedad de valor A a valor B en X segundos" y Godot lo hace suavemente. modulate:a controla la transparencia (0 = invisible, 1 = opaco). Es la forma sencilla de hacer fade-out sin necesidad de AnimationPlayer.
⚠️ set_deferred() es obligatorio aquí. No puedes desactivar una colisión mientras Godot está procesando una colisión (produciría un error). set_deferred() pospone el cambio al final del frame, cuando ya es seguro hacerlo. Recuérdalo siempre que modifiques colisiones dentro de callbacks de física.
5.5
Guarda como scenes/brick.tscn.
06

Crea la escena de la Pala (PadlePlayer)

Control de movimiento horizontal con suavizado

6.1
Nueva escena con nodo raíz CharacterBody2D.

Renómbralo PadlePlayer. Motion Mode: Floating (la pala solo se mueve en horizontal). Guarda como scenes/padle_player.tscn.

6.2
Añade los nodos hijos:
  • ColorRect → tamaño 128×32, posición (-64, -16), color cyan: #3bffff
  • CollisionPolygon2D → crea un polígono levemente redondeado en los extremos. Puntos: (-64,16), (-64,-8), (-8,-16), (8,-16), (64,-8), (64,16)

La forma de trapecio en la colisión hace que la bola rebote con ángulos diferentes según dónde impacte: más oblicuo en los extremos.

📦 PadlePlayer [CharacterBody2D · Motion: Floating] ← padle_player.gd ├── ColorRect [128×32 · color cyan] └── CollisionPolygon2D [forma trapezoidal]
6.3
Crea el script padle_player.gd y asígnalo. 
GDScriptextends CharacterBody2D

const SPEED = 600.0

func _physics_process(delta: float) -> void:
    # get_axis devuelve -1 (izq), 0 (nada) o 1 (der)
    var direction := Input.get_axis("ui_left", "ui_right")
    
    if direction:
        velocity.x = direction * SPEED
    else:
        # move_toward(valor_actual, destino, paso) — frena suavemente
        velocity.x = move_toward(velocity.x, 0, SPEED)
    
    move_and_slide()
💡 Input.get_axis() Es la forma moderna de leer dos teclas opuestas en Godot 4. Devuelve un valor continuo entre -1 y 1 — soporta también gamepad con sensibilidad analógica. Equivale a hacer izq - der pero más limpio.
💡 move_toward() Acerca un valor a otro sin pasarse. Usado aquí para que la pala frene gradualmente al soltar la tecla en lugar de parar en seco. Le pasamos la misma SPEED como paso, lo que hace que la pala se detenga en exactamente 1 frame — comportamiento responsivo pero sin tirones.
07

Monta la escena principal (Main)

Fondo, marcador, límites, pala, bola, cuadrícula de ladrillos y lógica de partida

7.1
Nueva escena con nodo raíz Node.

Renómbralo Main. Guarda como scenes/main.tscn y márcala como escena principal (Proyecto → Configuración → Application → Run → Main Scene).

7.2
Añade el fondo.

Añade un ColorRect como hijo. Anclado a toda la ventana (tamaño completo). Color: #0f0f0f (casi negro).

7.3
Añade el marcador de puntuación.

Añade un nodo Label, renómbralo LabelScore. Texto inicial: 000. Tamaño de fuente: 300. Color: blanco con alpha bajo (rgba(255,255,255,0.45)) para que quede como marca de agua de fondo. Posición centrada en la pantalla.

7.4
Añade los límites del campo de juego.

Añade un StaticBody2D llamado Limit. Dentro, añade un CollisionPolygon2D. Define un polígono en forma de U invertida que cubra las tres paredes (izquierda, superior, derecha) pero deje el fondo abierto (aquí detectaremos la pérdida de la bola).

Puntos del polígono: (0,720), (0,0), (1280,0), (1280,720), (1328,720), (1328,-40), (-48,-40), (-48,720)

7.5
Instancia la pala y la bola.

Clic derecho en Main → Instanciar escena hijo → padle_player.tscn. Repite para ball.tscn. Las posiciones iniciales las gestionará el script, no importa dónde las dejes ahora.

7.6
Añade los Markers de posición inicial.

Añade un nodo Node llamado Markets. Dentro, dos Marker2D:

  • MarkerBall → posición (640, 256) — dónde aparece la bola
  • MarkerPaddle → posición (640, 608) — dónde aparece la pala
💡 ¿Para qué sirve un Marker2D? Es un nodo sin forma ni lógica, simplemente marca una posición en la escena. Lo usamos para definir los puntos de inicio de la bola y la pala de forma visual, sin hardcodear coordenadas directamente en el código. Si quieres cambiar dónde empiezan, solo mueves el Marker en el editor.
7.7
Coloca la cuadrícula de ladrillos.

Añade un nodo Node llamado Bricks. Instancia brick.tscn repetidamente (48 instancias, 12 columnas × 4 filas). Separación entre ladrillos: 88 píxeles horizontalmente, 24 verticalmente. Primera fila en Y=56, primera columna en X=160.

💡 Puedes seleccionar un ladrillo ya colocado, copiarlo (Ctrl+D) y posicionarlo manualmente, o usa la herramienta Grid Snap del editor.

7.8
Añade el área de detección de bola perdida.

Añade un Area2D llamado AreaDown. Dentro, un CollisionShape2D con un RectangleShape2D de 1379×46 en posición (646, 760) — justo debajo del borde inferior de la ventana.

📦 Main [Node] ← main.gd ├── ColorRect [fondo negro] ├── LabelScore [Label · fuente 300px] ├── Limit [StaticBody2D · paredes] │ └── CollisionPolygon2D ├── PadlePlayer [instancia padle_player.tscn] ├── Ball [instancia ball.tscn] ├── Bricks [Node] │ ├── Brick × 48 instancias ├── Markets [Node] │ ├── MarkerBall [Marker2D · (640,256)] │ └── MarkerPaddle [Marker2D · (640,608)] └── AreaDown [Area2D · detector suelo] └── CollisionShape2D
7.9
Crea el script main.gd y asígnalo al nodo Main. 
GDScript# Nodo principal del juego — coordina bola, pala y puntuación
extends Node

func _ready() -> void:
    # Al cargar la escena, lanzamos la partida automáticamente
    play_game()

func _process(delta: float) -> void:
    # Actualizamos el marcador en cada frame
    $LabelScore.text = str(Global.score)

func play_game() -> void:
    # Coloca la pala y la bola en sus posiciones de inicio
    $PadlePlayer.position = $Markets/MarkerPaddle.position
    $Ball.position    = $Markets/MarkerBall.position
    # Arranca el movimiento de la bola
    $Ball.start()

# Señal del AreaDown: la bola cayó — reiniciamos la partida
func _on_area_down_body_entered(body: Node2D) -> void:
    play_game()
7.10
Conecta la señal del AreaDown.

Selecciona AreaDown → pestaña Nodo → señal body_entered → conecta a _on_area_down_body_entered en el nodo Main.

⚠️ El marcador no reinicia al perder la bola. Fíjate que play_game() no llama a Global.reset_score(). Esto es intencionado: en esta versión la puntuación es acumulativa y no se resetea al fallar. Si quieres que se reinicie, añade Global.reset_score() dentro de play_game().
08

¡Prueba el juego!

Lista de verificación y errores frecuentes

8.1
Pulsa F5 y comprueba:
  • ✅ La bola cae desde el centro al arrancar
  • ✅ La pala se mueve con ← → y con A/D
  • ✅ La bola rebota en las paredes y en la pala
  • ✅ Al tocar un ladrillo este se desvanece y suma punto
  • ✅ El número del marcador (fondo) aumenta
  • ✅ Si la bola cae por el fondo, se reinicia la posición
⚠️ Los ladrillos no desaparecen al ser tocados. Revisa que el nodo raíz de brick.tscn tiene asignado el grupo brick_group. Sin ese grupo, is_in_group("brick_group") devuelve false y no se llama a destroy_brick().
⚠️ La bola atraviesa la pala. Comprueba que el CollisionPolygon2D de la pala está correctamente definido y que la bola tiene su CollisionShape2D activo. Asegúrate también de que el Motion Mode de ambos CharacterBody2D está en Floating.
⚠️ Error en brick.gd: queue_free sin paréntesis. Si al romper un ladrillo Godot da un error, revisa finish_animation(). El código original tiene queue_free sin los paréntesis (). Debe ser queue_free() con paréntesis para llamar la función correctamente.
Resumen de Conceptos Trabajados

StaticBody2D brick.tscn, Limit Cuerpo físico inamovible; paredes y obstáculos

Tween brick.gd Animar propiedades de forma programática (fade-out del ladrillo)
set_deferred() brick.gd Modificar propiedades de física de forma segura fuera del ciclo de física
@tool circule.gd Ejecutar el script también en el editor, para previsualizar elementos visuales
_draw() circule.gd Dibujar formas geométricas por código sin imágenes externas
Marker2D Main → Markets Marcar posiciones de referencia en la escena sin lógica ni colisión
CollisionPolygon2D PadlePlayer, Limit Forma de colisión personalizada (polígono libre, no limitada a rect/círculo)

Actividades de Ampliación
★☆☆ FÁCIL

Cambia los colores

Asigna colores distintos a cada fila de ladrillos modificando el ColorRect de cada instancia en la escena Main.

★☆☆ FÁCIL

Reinicio de puntuación

Haz que la puntuación se ponga a 0 cuando la bola cae al fondo. Llama a Global.reset_score() dentro de play_game().

★★☆ MEDIO

Vidas del jugador

Añade una variable lives = 3 en Global. Cada vez que la bola cae, resta una vida. Muéstralas en pantalla y termina el juego al llegar a 0.

★★☆ MEDIO

Velocidad progresiva

Cada vez que se rompen 10 ladrillos, aumenta la SPEED de la bola un 10%. Añade un aviso en pantalla.

★★★ DIFÍCIL

Ladrillos de colores variables

Crea una variable @export var color: Color en brick.gd. Desde main.gd asigna colores distintos a cada fila por código en _ready().

★★★ DIFÍCIL

Power-ups

Al destruir ciertos ladrillos, crea un ítem que cae. Si la pala lo recoge, amplía su tamaño temporalmente usando un Tween sobre scale.x.
